1 What do we mean by thread?

1.1 Thread of execution in program

  • Compiler optimization
  • Requires the use of compilers that understand how to do this optimization and work with some sort of threading library like Intel's Thread Building Blocks (Intel compiler) or MPI.

1.2 Operating system thread

  • Scheduling unit in operating system
  • Used to be called "Lightweight process" in Unix
  • Applications have to use some sort of threading directive such as POSIX pthreads (as used in PARSEC) to implement parallel algorithms.

2 Are the SPEC CPU2006 benchmarks multi-threaded?

CONCLUSION
Individual SPEC CPU2006 benchmarks are single-threaded serial implementations.

2.1 Supporting documentation

  • From the CPU2006 web site (http://www.spec.org/cpu2006/docs/config.html) "Note: the SPEC CPU2006 benchmarks do not require multi-threading, and do not contain threading directives; but a compiler is allowed to try to automatically decompose threads, and there's nothing in the run rules to prohibit you from linking with your preferred version of a supported malloc library."
  • The shell scripts provided by SPEC to execute these benchmarks provide an option

2.2 What is the SPEC CPU2006 configuration?

  • The SPEC benchmarks use the GCC compiler.
    1. It does not any sort of automatic thread decomposition. Therefore, any sort of multithreading done in this case will require to use of threading directives.
    2. The benchmarks are built using the "-o2" option which only does very simple optimizations such as instruction scheduling. In particular, it does not any form of loop-rolling
